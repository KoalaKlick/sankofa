# Supabase Configuration

Supabase-specific configurations that Prisma cannot manage.

## Structure

```
supabase/
├── migrations/
│   ├── 00001_extensions.sql    # PostgreSQL extensions
│   └── 00002_rls_policies.sql  # Row Level Security policies
├── config.toml                 # Local development config
└── seed.sql                    # Seed data
```

## Database Schema

**Table schemas are managed by Prisma** in `prisma/schema/`.

```bash
# Create a migration
npx prisma migrate dev --name your_migration_name

# Apply to production
npx prisma migrate deploy
```

## RLS Policies

Row Level Security policies are in `00002_rls_policies.sql` because Prisma doesn't support RLS.

### Helper Functions

```sql
is_org_member(org_id)   -- Check membership
is_org_admin(org_id)    -- Check admin/owner role
is_org_owner(org_id)    -- Check owner role
get_org_role(org_id)    -- Get user's role
```

### Applying RLS Changes

1. Edit `migrations/00002_rls_policies.sql`
2. Run via Supabase CLI: `supabase db push`
3. Or run manually in Supabase Dashboard SQL Editor

## TypeScript Types

Types are generated by Prisma at `lib/generated/prisma`. Import:

```typescript
import { Event, Organization, Profile } from "@/lib/generated/prisma";
```
